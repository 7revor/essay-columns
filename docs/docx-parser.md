# DOCX 解析逻辑说明

## 概述

`src/utils/docxParser.ts` 负责将上传的 `.docx` 文件解析为结构化的作文数组。不依赖固定的文档格式，通过**重复行检测**自动推断作文边界。

## 数据结构

```ts
interface Essay {
  delimiter: string;    // 用于拆分作文的分隔标记行（可为空）
  paragraphs: string[]; // 作文正文段落
}
```

返回值：

```ts
{ essays: Essay[]; allLines: string[] }
```

- `essays` — 拆分后的作文数组
- `allLines` — 文档全部原始行（供后续模式检测使用）

## 解析流程

```
.docx 文件
  │
  ▼
mammoth.extractRawText()       ← 提取纯文本
  │
  ▼
按 \n 拆分为 allLines
  │
  ▼
findDelimiterPattern()         ← 重复行检测，推断分隔标记
  │
  ├─ 找到 delimiter → 按 delimiter 拆分
  │
  └─ 未找到 → 按空行拆分（fallback）
  │
  ▼
返回 { essays, allLines }
```

## 核心算法：重复行检测

### 原理

在一份包含多篇作文的文档中，每篇作文之间通常有某种重复出现的标记行，比如：

- `姓名：张三 班级：三年一班`（每篇都有，但内容不同 → 不适用）
- `--- 分隔线 ---`（完全相同 → 适用）
- `作文题目：我的梦想`（如果每篇都是同一题目 → 适用）
- 教师姓名、页码、装饰线等

算法寻找**完全相同的短行**中出现次数最多、间隔最均匀的那一行作为分隔标记。

### 算法步骤

```
1. 统计每行文本出现的位置列表（行号数组）
     │
2. 过滤候选：
     ├─ 出现次数 ≥ 3
     ├─ 长度 ≤ 50 字符
     └─ 相邻出现间隔平均值 ≥ 2 行
     │
3. 对每个候选计算评分：
     score = count × 1/(1 + cv)
     │
     ├─ count = 出现次数（越多越好）
     └─ cv = 间隔的变异系数（越小 = 越均匀 = 越好）
     │
4. 取评分最高的行作为 delimiter
```

### 评分公式

```
cv = σ / μ    （变异系数 = 标准差 / 均值）
score = count × 1/(1 + cv)
```

- **count** — 重复次数越多，越可能是分隔标记
- **cv** — 间隔越均匀（cv 趋近 0），越可能是分隔标记而非正文巧合
- 两者相乘，兼顾数量和分布

### 示例

假设文档有 100 行，某行出现在第 1、21、41、61、81 行：

```
count = 5
gaps = [20, 20, 20, 20]
μ = 20, σ = 0, cv = 0
score = 5 × 1/(1+0) = 5.0    ← 非常高，几乎一定是分隔标记
```

另一行出现在第 3、7、50、90 行：

```
count = 4
gaps = [4, 43, 40]
μ = 29, σ ≈ 17.9, cv ≈ 0.617
score = 4 × 1/(1+0.617) ≈ 2.47    ← 较低，间隔不均匀
```

## 拆分逻辑

### 有 delimiter 时

```
逐行扫描
  │
  ├─ 行 === delimiter → flush 当前段落为一篇作文，重置
  ├─ 空行 → 跳过
  └─ 其他 → 追加到当前段落缓冲区
  │
遍历结束 → flush 最后一篇
```

支持两种 delimiter 位置：
- **delimiter 在作文前面**（header 风格）：第一个 delimiter 之前的内容如果非空也会被收录
- **delimiter 在作文后面**：最后一个 delimiter 之后的内容也会被收录

### 无 delimiter 时（fallback）

当没有找到任何重复行（比如文档只有一篇作文，或格式完全自由），按空行拆分：

```
连续非空行 → 一篇作文
空行 → 分隔
```

## 兼容性

| 文档格式 | 处理方式 |
| --- | --- |
| `姓名：xx 班级：xx` 开头的传统格式 | 该行会被检测为高频重复模式，自动作为 delimiter |
| `--- 分隔线 ---` 分隔 | 同上 |
| 教师信息/页码等重复行分隔 | 同上 |
| 无任何重复标记 | fallback 到空行拆分 |
| 单篇作文 | 整体作为一篇 |

## 下游处理

解析后的 `essays` 传入 `patternDetection.ts`：

- `delimiter` 字段作为"分隔标记"模式展示给用户，可勾选是否在输出中保留
- 进一步检测作文段落中的重复内容（跨作文出现的相同段落）
- `filterEssay()` 根据用户选择过滤 delimiter 和重复段落，生成最终的 `EssayContent`
